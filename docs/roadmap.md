# 路线图

## 阶段 1：工作区落地

- 完成多 mod 工作区目录结构。
- 记录游戏根目录路径。
- 建立同步、安装、清理、打包脚本的安全占位版本。

## 阶段 2：确认参考文件来源

- 从游戏目录复制需要的 DLL 或资源到 `references/`。
- 记录 arm64 与 x86_64 目录中哪些文件与 mod 开发相关。
- 形成最基础的 API 笔记与类映射草稿。

## 阶段 3：首个真实 mod 骨架

- 从 `mods/00-template-mod/` 复制出第一个正式 mod。
- 确认 manifest 格式、入口方式和最小可运行文件集合。
- 完成首次可重复导出到 `exports/release/`。

## 阶段 4：安装与验证

- 确认游戏目录中的真实安装位置。
- 完善 `install-mod.sh` 的目标子目录和校验逻辑。
- 建立最小回归测试流程：安装、启动、观察、回滚。

## 阶段 5：首个可加载 mod

- 实现最小功能。
- 记录加载成功所需条件。
- 固化模板与文档，支持第二个 mod 复用。
