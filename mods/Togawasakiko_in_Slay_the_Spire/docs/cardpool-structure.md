# 卡池结构规划

日期：2026-03-24

## 一 规划范围

本文件讨论的是角色的正常卡池规划，重点是普通 / 非凡 / 稀有三档的分布建议。

当前已确认：

- 正常卡池总量：`50`
- 压力衍生牌不计入这 `50` 张

补充说明：

- 起始牌与正常卡池属于同一套角色牌系。
- 原版角色在稀有度上还存在 `Basic` 层。
- 因此本文件聚焦的是 `Common / Uncommon / Rare` 三档分布，不代表把起始牌当成独立系统。

## 二 品质分布建议

结合本地原版角色卡池证据，当前冻结工作版继续采用以下结构规划：

- 普通品质：`12`
- 非凡品质：`22`
- 稀有品质：`16`

合计：`50`

## 三 原版角色对照依据

本轮直接参考了本地 IL 资料中原版角色的 `GenerateAllCards()` 与各卡构造 rarity：

- `IroncladCardPool.GenerateAllCards()`：`87` 张
- `SilentCardPool.GenerateAllCards()`：`88` 张
- `DefectCardPool.GenerateAllCards()`：`88` 张

其中三名原版角色的整体分布高度接近：

- Ironclad：
  - `Basic 3`
  - `Common 20`
  - `Uncommon 36`
  - `Rare 26`
  - `Ancient 2`
- Silent：
  - `Basic 4`
  - `Common 20`
  - `Uncommon 36`
  - `Rare 26`
  - `Ancient 2`
- Defect：
  - `Basic 4`
  - `Common 21`
  - `Uncommon 36`
  - `Rare 25`
  - `Ancient 2`

当前从这些原版角色可以直接看出两点：

- `Uncommon` 才是最大头，不是 `Common`
- `Rare` 在 STS2 当前角色卡池中的占比也明显高于 STS1 直觉

## 四 为什么采用这版比例

把原版三角色的 `Common / Uncommon / Rare` 近似缩放到 `50` 张后，得到的中心区间约为：

- `Common`：约 `12`
- `Uncommon`：约 `22`
- `Rare`：约 `16`

因此当前推荐 `12 / 22 / 16`，而不再使用上一轮的 `22 / 17 / 11`。

## 五 当前功能层面的建议

当前不直接给出单卡，但建议后续按以下方向配比：

- 普通品质：
  - 负责最稳定的施压、基础控制、防御过渡、低门槛压力消费
  - 承担最清楚的基础压力教学
- 非凡品质：
  - 负责更明显的机制联动、节奏拉开与牌序强化
  - 作为整套角色中最厚的一层，承接压力与原创 debuff 的中段联动
- 稀有品质：
  - 负责高上限 payoff、少量处决、整套体系的尖顶体验
  - 承担更高价值的压力兑现与少量处决设计

## 六 关于 Basic 层的当前处理

当前不建议把起始牌另写成独立体系。

更贴近原版的写法应是：

- 起始牌属于同一套角色牌系
- 只是它们大概率会落在 `Basic` 层
- 而本文件当前讨论的 `50` 张，是 `Common / Uncommon / Rare` 规划层

## 七 当前不要写死的内容

以下内容当前不应提前冻结：

- 各品质内的具体功能牌数量
- `Basic` 层最终是否保持 `10` 张起步配置不变
- 攻击 / 技能 / 能力牌的最终比例
- 处决类卡在稀有池中的最终占比
- 是否需要少量“反向围绕我方压力”的分支牌
- 是否需要在更后期额外讨论 `Ancient` 层

## 八 当前冻结边界

- 当前这套 `12 / 22 / 16` 作为 T2 收尾阶段的冻结工作版分布保留。
- 本轮不再改动这一比例。
- 后续若进入实现验证阶段发现需要微调，应在后续线程集中处理，而不是在 T2 文档里继续漂移。
